**Part A: Create CONTRIBUTING.md in repository root**

Create CONTRIBUTING.md with these sections:

1. **Header**
   - Title: "Contributing to Notiva"
   - Welcome message for contributors

2. **Getting Started**
   - Link to docs/SETUP.md for environment setup
   - Link to docs/ARCHITECTURE.md for understanding codebase
   - Fork and clone instructions

3. **Development Workflow**
   - Create feature branch from master
   - Make changes following code style
   - Test locally (link to docs/TESTING.md)
   - Commit with conventional messages
   - Push and create PR

4. **Code Style Guide**
   - Java conventions (4-space indentation, braces on same line)
   - Naming conventions (camelCase methods, PascalCase classes)
   - Documentation expectations (Javadoc on public methods)
   - Import ordering (Android, Java, project)
   - Android-specific patterns (use Android Log, avoid System.out)

5. **Commit Message Convention**
   - Based on conventional commits (observed from git log)
   - Format: `type(scope): description`
   - Types: feat, fix, docs, refactor, test, chore
   - Scope: optional, area of codebase (e.g., reminders, notifications)
   - Examples from actual git history:
     - `docs(09-01): create comprehensive UI documentation`
     - `feat: add reminder recurrence support`
   - Keep subject under 72 characters
   - Use imperative mood ("add" not "added")

6. **Pull Request Process**
   - Fill out PR template (if exists) or describe changes
   - Reference any related issues
   - Ensure tests pass
   - Wait for review
   - Address feedback
   - Merge after approval

7. **Documentation Contributions**
   - Documentation location: docs/ folder
   - Format: Markdown with Mermaid diagrams
   - Style: Developer-focused, practical examples
   - Link to existing docs for pattern reference
   - Update docs/README.md navigation if adding new docs

8. **Questions and Help**
   - Where to ask questions (Issues)
   - How to report bugs
