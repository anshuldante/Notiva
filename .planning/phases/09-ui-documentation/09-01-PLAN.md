  <action>
Create comprehensive UI documentation following the established Markdown + Mermaid pattern:

**1. Overview Section**
- Brief intro explaining the 2-screen architecture
- Screens inventory table with screen name, activity, layout, and purpose

**2. Screen: MainActivity**
Document the reminder list screen:
- Purpose: Display all reminders in a scrollable list
- Layout file: activity_main.xml
- Key components:
  - RecyclerView (am_rv_reminders) - displays reminder cards
  - TextView (am_tv_no_reminders) - empty state message
  - FloatingActionButton (am_fab_add_reminder) - create new reminder
- ViewModels injected: GetAllRemindersViewModel, ReminderDmlViewModel
- User interactions: tap card (edit), swipe (delete), tap FAB (create), toggle switch (enable/disable)
- Permission checks on launch: exact alarm, notifications

**3. Screen: UpsertReminderActivity**
Document the create/edit screen:
- Purpose: Create new or edit existing reminders
- Layout file: activity_upsert_reminder.xml
- Key components organized by MaterialCardView sections:
  - Time Picker card (card_time): TimePicker for start time
  - Date card (card_date): TextView + ImageView calendar picker
  - Name card (card_name): TextInputEditText for reminder name
  - Repeat switch card (card_repeat): SwitchCompat for recurrence toggle
  - Recurrence card (card_recurrence): Spinner, EditText, end date/time pickers
  - Button bar: Cancel and Save MaterialButtons
- ViewModel injected: ReminderDmlViewModel
- Mode detection: checks for REMINDER_ID extra to determine create vs edit

**4. Component: ReminderItemAdapter**
Document the RecyclerView adapter:
- Purpose: Bind ReminderModel to rv_item_reminder layout
- Key ViewHolder components:
  - rir_tv_alarm_name - reminder name (hidden if empty)
  - rir_tv_next_occurrence - formatted next trigger time
  - rir_tv_summary - recurrence summary (hidden if non-recurring)
  - rir_sw_active - toggle switch
- Visual states: normal (alpha 1.0) vs disabled/expired (alpha 0.5, strikethrough)
- Interactions: item click (edit), switch toggle (update status)

**5. Layout Patterns**
Document reusable patterns:
- MaterialCardView with 16dp corner radius for grouping
- ConstraintLayout for flexible positioning
- CoordinatorLayout for FAB behavior
- ScrollView wrapping for form screens
- Consistent padding (16dp) and margins

**6. Navigation Flow Diagram**
Create Mermaid flowchart showing:
- App launch -> MainActivity
- MainActivity FAB tap -> UpsertReminderActivity (create mode)
- MainActivity item tap -> UpsertReminderActivity (edit mode, with extras)
- UpsertReminderActivity save/cancel -> MainActivity
- Permission dialogs (exact alarm settings, notification permission)

**7. Intent Extras Reference**
Document the constants used for passing data:
- REMINDER_ID, REMINDER_NAME, REMINDER_ACTIVE
- REMINDER_START_TIME, REMINDER_END_TIME
- REMINDER_RECURRENCE_DELAY, REMINDER_RECURRENCE_TYPE
- Source: ReminderConstants utility class

**8. Adding a New Screen**
Guide for developers:
- Create Activity class extending AppCompatActivity
- Add @AndroidEntryPoint for Hilt injection
- Create layout XML in res/layout/
- Register in AndroidManifest.xml
- Use Intent to navigate, pass data via extras
- Follow MaterialCardView grouping pattern
